# sumi.plan.md
> Project: 숨이 (Sumi) Growth Artifact Cloud Page + SEO
> Goal: Growth Artifact(기록) 페이지를 정적 사이트로 배포(Cloudflare Pages)하여 Google 검색 인덱싱 및 장기 SEO 자산으로 축적한다.

---

## 0) 성공 기준 (Definition of Done)
- [ ] `https://<domain>/` 랜딩 페이지가 배포되어 접속 가능
- [ ] `https://<domain>/journal` 목록 페이지 + `https://<domain>/journal/<slug>` 상세 페이지 동작
- [ ] `sitemap.xml` 자동 생성/제공 + GSC 제출 가능
- [ ] `robots.txt` 제공
- [ ] 각 상세 페이지에 `title`, `description`, `canonical`, `og:*` 포함
- [ ] Google Search Console에서 URL 검사 시 “인덱싱 가능” 상태 확인
- [ ] 최소 10개의 Growth Artifact 글이 게시되어 링크 구조(내부링크)가 연결됨
- [ ] (선택) GA4 또는 PostHog 중 1개 이상으로 페이지뷰/유입 추적 가능

---

## 1) 제품/콘텐츠 원칙
### 1.1 브랜드/톤
- 브랜드명: **숨이**
- 핵심 문장: **“잘 쉬는 몸을 만드는 시간”**
- 의료/치료/교정/성장 촉진 결과 보장 금지
- ‘기록’ 중심: 설명/전후비교/전문가 주장 최소화

### 1.2 콘텐츠 포맷(권장)
- 각 글은 300~600자 중심, H2 1~3개
- 본문 하단 CTA는 “소개 보기” 수준(예약 강요 X)

---

## 2) 기술 선택 (권장 스택)
- Framework: **Astro (SSG)** (권장)  
  - 대안: Next.js SSG / VitePress
- Hosting: **Cloudflare Pages**
- Content: **Markdown/MDX** (Git 기반)
- Styling: Tailwind(선택) 또는 기본 CSS
- Analytics: GA4(선택) 또는 PostHog(선택)
- SEO: sitemap, robots, canonical, og, structured data(선택)

---

## 3) 정보구조(IA) & 라우팅
### 3.1 Routes
- `/` : 랜딩 (브랜드 소개 + 3세대 섹션 + journal로 이동)
- `/journal` : Growth Artifact 리스트(카테고리/태그 필터)
- `/journal/<slug>` : 글 상세
- `/about` (선택): 철학/경계(치료 아님)/문의
- `/privacy` (선택): 개인정보 처리방침(문의 폼/분석도구 사용 시)
- `/terms` (선택): 이용약관(필요 시)

### 3.2 URL 규칙
- slug: kebab-case(한글도 가능하나 권장은 영문/로마자 or 한글 slug를 자동 변환)
- 예: `/journal/well-rest-needs-no-explanation`
- 내부링크: 글 본문 하단에 “관련 글 3개” 자동 노출

---

## 4) 콘텐츠 모델 (Frontmatter)
Markdown 파일 예시:

```yaml
---
title: "잘 쉬는 몸은 설명이 필요 없다"
description: "숨이는 잘 쉬는 몸을 만드는 라이프 이완 케어입니다."
date: "2026-02-10"
category: "women" # women | kids | parents
tags: ["종아리", "이완", "쉼", "숨"]
slug: "well-rest-needs-no-explanation"
draft: false
---
````

필수: `title`, `description`, `date`, `category`, `slug`, `draft`
선택: `tags`, `coverImage`, `ogImage`

## 5) SEO 설계

### 5.1 Meta

각 페이지:

* `<title>${title} | 숨이</title>`
* `<meta name="description" content="${description}">`
* `<link rel="canonical" href="${siteUrl}${path}">`
* OG: `og:title`, `og:description`, `og:type=article`, `og:url`
* Twitter card(선택)

### 5.2 robots & sitemap

* `robots.txt`:

  * `User-agent: *`
  * `Allow: /`
  * `Sitemap: https://<domain>/sitemap-index.xml` (또는 sitemap.xml)
* sitemap:

  * Astro plugin 또는 직접 생성
  * draft 글 제외
  * journal list 포함

### 5.3 Structured Data(선택)

* Article JSON-LD (title, datePublished, author="숨이", description, url)

### 5.4 내부링크 전략

* `/journal`에서 카테고리별 섹션 제공
* 상세 페이지 하단 “관련 글 3개” (같은 category 우선 + tag 교집합 우선)

## 6) 분석(Analytics) 최소 구현

* (선택) GA4: Cloudflare Pages 환경변수로 measurement id 주입
* (선택) PostHog: minimal snippet + pageview 이벤트
* 이벤트 최소:

  * `page_view`
  * `cta_click` (e.g., "about", "journal_list", "contact")

## 7) 배포/도메인/GSC 연결

### 7.1 Cloudflare Pages

* GitHub repo 연결
* Build command:

  * Astro: `npm run build`
* Output dir:

  * Astro: `dist`
* Env:

  * `SITE_URL=https://<domain>`

### 7.2 도메인

* `<domain>`을 Cloudflare DNS에 연결
* HTTPS 자동 적용 확인

### 7.3 Google Search Console

* 도메인 속성으로 등록(권장)
* DNS TXT verification
* sitemap 제출
* 주요 URL 검사(홈/저널/상세 1개)

## 9) 초기 콘텐츠 백로그 (제목만)

### Women

* 잘 쉬는 몸은 설명이 필요 없다
* 하루가 끝나도 다리가 쉬지 않는 이유
* 잠들기 전, 종아리부터 풀리는 순간
* 오늘은 운동보다 이완이 먼저였다

### Kids

* 아이 키보다 먼저 봐야 했던 신호
* 잘 자는 날과 못 자는 날의 차이
* 성장기 아이 다리는 늘 긴장해 있다

### Parents

* 강하지 않아도 충분했던 손
* 병원 말고 사람이 필요한 순간
* 아프지 않은 손이 먼저였다
